## Overview
This project features a modern, interactive weather forecast web application developed by **Hsuan-Lin** as part of the **Internet of Things (IoT)** course curriculum. The project focuses on integrating data fetching from government open data platforms, data processing, and front-end visualization using Streamlit, highlighting skills in Python, API integration, and geographic data representation.

---

## Completed Tasks

### 1. Data Fetching & Processing (`fetch_weather.py`)
* **CWA API Integration:** Developed a robust script to autonomously fetch 7-day weather forecast data from the Taiwan Central Weather Administration (CWA) API (endpoint `F-D0047-091`), successfully handling SSL verification challenges.
* **Data Parsing & Aggregation:** Designed algorithms to parse complex JSON responses, extracting `MaxTemperature` and `MinTemperature` data. The system intelligently aggregates county-level data into 6 major target regions (北部, 中部, 南部, 東北部, 東部, 東南部) and computes the average daily temperatures.
* **Data Storage:** Exported the parsed and structured regional data into a clean `weather_data.csv` for downstream consumption by the web application.

### 2. Interactive Web Application (`app.py`)
* **Streamlit Framework:** Built a fast, responsive single-page application featuring a clear left-right split-screen UI layout.
* **Geospatial Visualization (Folium):** Integrated a dynamic map of Taiwan centered automatically with region-specific `CircleMarkers`. Implemented conditional color coding (Blue: <20°C, Green: 20-25°C, Yellow: 25-30°C, Red: >30°C) based on average temperatures, alongside detailed popups featuring min/max/avg stats.
* **Interactive Data Filtering:** Added a dynamic date selector dropdown that enables users to filter the visual map and the corresponding data table in real-time, delivering a seamless AIoT-style data consumption experience.

### 3. Environment & Deployment Setup
* **Dependency Management:** Configured a dedicated Python virtual environment and formalized project requirements (`requests`, `pandas`, `streamlit`, `folium`, `streamlit-folium`) allowing for quick reproduction of the working state.
* **Version Control Integration:** Established local Git version control and pushed the complete structured repository to GitHub, ready for further deployment or integration.

## File Structure
* `app.py`: The core Streamlit web application defining UI layout and Folium map rendering logic.
* `fetch_weather.py`: The backend data ingestion script to query the CWA API and process JSON into CSV.
* `requirements.txt`: Defines all library dependencies needed to execute the application.
* `weather_data.csv`: Locally cached dataset generated by the fetcher script.
* `README.md`: Project documentation and technical overview (this file).

## How to Run Locally

1. Create and activate a Python virtual environment:
   ```bash
   python -m venv venv
   # On Windows:
   .\venv\Scripts\activate
   ```
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run the Streamlit application:
   ```bash
   streamlit run app.py
   ```
   *(Note: The app will automatically fetch the latest data from the CWA API on first launch if `weather_data.csv` is not present.)*
